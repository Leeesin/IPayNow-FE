# 分页组件

## 解决了什么问题？
::: tip
- 大部分业务的分页逻辑是一致的,没必要每个页面写一遍，封装成组件，能减少代码出错概率，也能提升业务开发笑率。
:::


## 注意事项
::: warning 
- 可能每个项目的接口类型不一致，需要修改部分接口相关的源码
:::

## 源码

``` vue
<template>
  <div class='v-pagination'>
    <el-pagination @size-change="onSizeChange"
                   @current-change="onCurrentChange"
                   :current-page="pagination.pageNum"
                   :page-sizes="pagination.sizes"
                   :page-pageSize="pagination.pageSize"
                   layout="total, sizes, prev, pager, next, jumper"
                   :total="pagination.total">
    </el-pagination>
  </div>
</template>

<script>
export default {
  props: {
    //tableData
    value: {
      type: Array,
      required: true
    },
    //查询条件
    queryObj: {
      type: Object,
      required: true
    },
    //控制父表格loading
    loading: {
      type: Boolean,
      required: true
    },

    funcode: {
      type: String,
      required: true
    }
  },
  data() {
    return {
      pagination: {
        pageNum: 1,
        pageSize: 10,
        sizes: [10, 20, 30, 40],
        total: 0
      }
    };
  },
  methods: {
    //提供给父组件进行查询操作
    //页码重置为1 并且请求第一页数据
    async init() {
      this.pagination.pageNum = 1;
      this.update();
    },
    //父组件可以用此方法 刷新当前页列表数据 
    async update() {
      const payload = await this.search();
      this.$emit("input", payload);
    },

    async search() {
      this.$emit("update:loading", true);
      const { data } = await this._http({
        funcode: this.funcode,
        ...this.queryObj,
        pageNum: this.pagination.pageNum,
        pageSize: this.pagination.pageSize
      });
      this.pagination.total = data.total;
      this.$emit("update:loading", false);
      return data.rows;
    },

    async onSizeChange(pageSize) {
      this.pagination.pageSize = pageSize;
      this.update();
    },
    async onCurrentChange(pageNum) {
      this.pagination.pageNum = pageNum;
      this.update();
    }
  },
  mounted() {
    this.update();
  },
};
</script>
```
