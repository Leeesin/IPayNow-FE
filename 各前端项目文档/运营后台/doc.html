# 运营后台目录结构

注意

  • 线上分支为develop,开发时需从此分支拉取代码。
  • 测试分支为feature/opt-test
  • common/js/directive.js: 用于存放自定义组件，可以来这里看组件的api使用方法。
  • common/views/menu.html: 新加的页面需要到这里增加菜单
  • common/controllers/controllers.js:二级域名定义

注意

开发项目时切记运行 gulp run，这会将路由编译到all.route.config.js,否则需要在all.route.config.js内创建路由， 因为该文件内代码较多，不建议直接在此文件新增路由。
